Para utilizar a versão online do Game Design Canvas clique em "File", então em "Make a Copy". Isso vai criar um arquivo no seu drive. Agora você pode compartilhar com os outros membros do grupo clicando em "Share", que é um botão azul no canto superior direito. Acho que precisa ter uma conta do Gmail para utilizar assim. Assim todo o grupo mexe no mesmo arquivo. Cada um pode ter um arquivo separado e também um outro para o grupo todo.

O que cada lição ensina:

Módulo 1

  1. Banda de Rock
  2. Perdido no Espaço
  3. Caça Fantasmas:
  4. Robô Falante: Trabalhar com entradas do jogador e interagir com elas. O jogador pode digitar algo que será usado dentro do jogo.
  5. Lousa Mágica: Alterações no jogo através de mensagens que um elemento envia para o resto. Por exemplo: clico em uma opção e ela muda algo no meu personagem. Pode ser usado para fazer uma tela que você seleciona características do personagem, como cor da camiseta, pele, sapato e etc.
  6. Corrida de Barco: ensina a adicionar cronômetro no jogo que decresce até que o objetivo seja alcançado. Fazer um personagem que segue o mouse e se modifica quando bate em algo. Adicionar obstáculos e outros elementos do cenário que interagem com o seu personagem. Como fazer mudança de níveis/fases.

Módulo 2

  1. Jogo da Memória: Ensina como usar listas de variáveis. Como salvar o maior placar de um jogador para comparar com os outros jogadores.
  2. Esquivabol: Um jogo de plataforma com inimigos e obstáculos.
  3. Jogo da Tabuada: Ensina como a fazer uma tela inicial para o jogo, com opção para iniciar ou fazer o tutorial.
  4. Capturando Bolinhas: Usando cores para modificar a interação com os objetos do mapa.
  5. Guerra dos Clones: Adicionar inimigos que lancem algo para destruir o seu personagem. Fazer uma tela de Game Over.
  6. Um jogo que você pode mudar de cenário através de portas. Coletar itens e interagir com personagens.

Scratch

Construindo um jogo em conjunto no Scratch

Bom, serão várias pessoas trabalhando em um mesmo projeto, certo? Mas não faremos uma conta para todos de um grupo compartilharem, isso não dá muito certo. Ao invés, vamos seguir as dicas do pessoal do Scratch! Então, com base nelas, faremos da seguinte forma para trabalhar em conjunto em um mesmo projeto:

  1. Criem uma conta do Scratch para cada membro do grupo (fizemos na primeira aula, quem não tiver é só criar ou me chamar para ajudar!
  2. Decidam entre vocês quem será o líder do projeto. Lembrem-se, líder não é quem manda e nem o único responsável, mas quem organiza!
  3. O líder criará em sua conta o projeto oficial. O que é o projeto oficial? Será aquele com a versão final. Nele será integrado aos poucos o que cada um da equipe vai fazendo. O líder será o responsável por adicionar os novos trabalhos neste projeto.
  4. Cada pessoa do grupo, inclusive o líder, devem criar uma cópia do projeto para sua própria conta. Isso pode ser feito acessando a página do projeto oficial, clicando no botão "ver interior" e depois em "remix", no canto superior direito.
  5. Tendo uma cópia para cada um, a pessoa vai trabalhar em um determinado aspecto do projeto, uma tarefa bem pequena e específica, como adicionar um inimigo, fazer a contagem de pontos e etc. Tendo terminado, o líder deverá adicionar o código, arquivos externos (sons, figuras e etc) ao projeto oficial e os dois, líder e pessoa que implementou, devem verificar se está funcionando como deveria.
  6. Cada vez que algo novo é adicionado no projeto, talvez seja a hora de criar uma nova cópia do projeto oficial para a sua conta do Scratch. Basta clicar novamente em "remix". O Scratch não vai colocar essa cópia por cima da antiga. Se você clicar 10x em "remix", terá 10 projetos copiados para a sua conta!
  7. Tudo isso vai ficar mais claro quando estiverem trabalhando! :)

Boas práticas de trabalho em conjunto

  1. Cada membro possui uma categoria para se dedicar mais
  2. O grupo todo discute sobre o projeto com frequência
  3. Decisões conjuntas sobre o rumo do projeto, com possibilidade de votação
  4. O grupo possui um líder, que se responsabiliza em conferir se as pessoas estão cumprindo suas tarefas
  5. Um meio de comunicação organizado, estilo fórum, como um grupo do Facebook e onde todos estejam
  6. Quem atualiza o que, controle das mixagens e etc, para que o projeto não fique bagunçado

Tarefas comuns no desenvolvimento conjunto

  • Programação: construir os bloquinhos no Scratch!
  • Arte: fazer o desenho dos sprites (fantasias)
  • Sons: criar, encontrar ou editar efeitos sonoros para projeto. Também compor músicas para o jogo.
  • Idéias: pensar em novos elementos, estruturas, personagens, fases e etc
  • Teste: jogar o que já está feito e anotar e informar todos os erros que encontrar no jogo
  • História: escrever e elaborar a história do jogo. Algo para ser incluido no jogo, ou como documentação
